Концептуальная задача - разработать систему позволяющую вести учёт выявленных на оборудовании тепловых сетей
повреждений. Данные о повреждения должны храниться цетрализованно на сервере базы данных и могут быть доступны
для чтения всем авторизованным пользователям с правами "Гостя", для чтения и модификации авторизованным пользователям 
с правами "Диспетчер" и "Администратор". Приложение должно иметь web-серверную архитектуру. Доступ к клиентской части 
приложения должен осуществляться через web-браузер (IE). Сервенрная часть приложения также должна осуществлять обмен 
данными с установленным на предприятии сервером ГИС Zulu 7.0 для внесения и модификации данных о повреждениях на
слое "Дефекты". Карта с внесёнными сведениями о повреждениях также должна быть доступна в клиентской части приложения
с применением технологии "ActiveX".

Требования к системе:
1. Аутентификация пользователей при входе в систему
2. Для пользователь "Гость"
2.1. отображение перечня (списка) всех выявленных повреждений
2.2. возожность сортировки и филтрации списка повреждений
2.3. отображение актуальных сведений о повреждениях на карте (ГИС Zulu)
2.4. вызов подробных сведений о выделенном повреждении
3. Для пользователя "Диспетчер"
3.1. все требования для пользователя "Гость"
3.2. возможность внесения новых данных о повреждении оборудования
3.3. возможность модификации и удаления уже внесённых данных
4. Для пользователя "Администратор"
4.1. все предыдущие функции
4.2. создание и изменение учётных записей пользователей
4.3. настройка конфигурации системы
